# CLAUDE.md — AI Development Guide for PMP Trainer

## Project Overview

PMP Exam Trainer is a single-file HTML application for practicing PMP certification exam questions. The entire app (HTML + CSS + JS + data) lives in `index.html`. No build tools, no bundler, no framework.

## Architecture

### Single-file design

Everything is in `index.html`:
- CSS styles in a `<style>` block
- Question data in a `<script>` block (JSON assigned to JS variables)
- Application logic in a second `<script>` block

### Data structure

Three global JS variables hold all data:

```
ALL_QUESTIONS  — Array of 1387+ question objects
DD_QUESTIONS   — Array of 15 drag-and-drop matching exercises
TOPIC_CODES    — Array of integers mapping each question index to a PMP domain
```

Question object shape:
```js
{
  id: 1,                          // Unique integer ID
  type: "single_choice",          // or "multi_select"
  question: "Question text...",
  options: { "A": "...", "B": "...", "C": "...", "D": "..." },
  correct: "A",                   // Single key
  correct_list: ["A"],            // Array of correct keys (length > 1 for multi-select)
  community_votes: { "A": 99 },   // Optional vote percentages
  imported: false                  // true for user-imported questions
}
```

Drag-and-drop object shape:
```js
{
  id: 9001,
  type: "drag_drop",
  question: "Match each item...",
  drag_items: ["Item 1", "Item 2"],
  drop_zones: ["Zone A", "Zone B"],
  correct_mapping: { "Item 1": "Zone A", "Item 2": "Zone B" },
  explanation: "HTML explanation text"
}
```

### State management

All state is in plain JS variables, persisted to localStorage:

- `session` (key: `pmp_v4`) — answered questions, correct/wrong tracking, current index, mode
- `importedQuestions` (key: `pmp_imported`) — user-imported question arrays
- `aiCache` — in-memory only, explanation HTML cache (not persisted)
- `ddState` — in-memory only, drag-and-drop placement state

### Topic classification

Questions are classified into 11 PMP domains via the `TOPIC_CODES` array:

| Code | Domain |
|------|--------|
| 0 | Agile / Scrum |
| 1 | Earned Value Management |
| 2 | Risk Management |
| 3 | Stakeholder Management |
| 4 | Schedule Management |
| 5 | Quality Management |
| 6 | Procurement Management |
| 7 | Resource & Team Management |
| 8 | Integrated Change Control |
| 9 | Communications Management |
| 10 | General PMP Best Practice |

Topic codes are pre-computed during build. Imported questions don't have topic codes and fall back to code 10 (General).

## Critical Constraints

### No template literals
All JS uses string concatenation (`'...' + var + '...'`). Never use backtick template literals — the file is generated by a Python build script and backticks cause escaping conflicts.

### No Unicode in JS strings
All emoji and special characters in JS string literals must use HTML entities (`&#128161;` not `💡`). Unicode chars inside JS strings cause `Invalid or unexpected token` errors depending on file encoding.

### No inline event handlers with quotes
Never write `onclick="fn('arg')"` inside JS-generated HTML strings. Quote nesting breaks. Use `addEventListener` after rendering, or pass only numbers/simple values: `onclick="fn(123)"`.

### No multiline strings
JS strings must be single-line. Never put actual newline characters inside quotes. Use `'line1' + '\n' + 'line2'` or set content via `element.textContent`.

### No external API calls
The app runs in environments that block outbound requests (CORS, CSP). All explanations are generated locally from `TOPIC_META` data. No `fetch()` calls.

### localStorage keys
- `pmp_v4` — session state
- `pmp_imported` — imported questions

Do not change key names without migration logic, or users lose their progress.

## Common Tasks

### Adding new drag-and-drop exercises

Edit the `DD_QUESTIONS` array in the build script (`build_final.py`) or directly in `index.html`. Follow the existing object shape. Keep `id` in the 9000+ range.

### Adding new PMP domains

1. Add entry to `TOPIC_META` array in JS
2. Add corresponding detection logic in the Python build script's `get_topic_code()` function
3. Regenerate `TOPIC_CODES` array

### Modifying the import parser

The `parseQuestions()` function in JS handles user-uploaded JSON. It supports:
- `options` as array of `"A. text"` strings
- `options` as object `{A: "text"}`
- `answer` or `correct` field containing `"A. full text"` or just `"A"`
- Reference stripping: `[16]`, `[16, 17]` removed from question and answer text

### Rebuilding from data

The Python build script at `build_final.py` (not in repo, used during development):
1. Reads `pmp_typed.json` (classified questions)
2. Reads `dd_v2.json` (drag-drop exercises)
3. Reads `topic_codes.json` (domain classification)
4. Writes everything into a single `index.html`

## Testing

Open `index.html` in a browser and verify:
1. Questions render and navigation (prev/next/jump) works
2. Single-choice selection highlights correct/wrong
3. Multi-select allows picking multiple options and submitting
4. Explain button shows domain-based explanation
5. Drag-and-drop items can be dragged and checked
6. Import accepts a JSON file and adds questions
7. Dashboard shows correct statistics
8. localStorage persistence survives page reload
